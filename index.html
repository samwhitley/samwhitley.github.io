  • Instant Messaging

    2020
    • Demo
    • Source Code

    This is a simple mobile-friendly instant messaging app. It has separate servers for its API and client interface. The API server is built in Express and hosts the existing user messages. The client calls the API and renders the existing messages server-side before returning the page in the browser. I made this to demonstrate server-side rendering, a vital SEO technique for web apps that build interfaces via JavaScript.

    Skills used:

    • HTML
    • JavaScript: Node, Express, ES6, Gulp
    • CSS: Sass

  • SVG Hangman

    2014
    • Demo
    • Source Code

    This is a Hangman game - you're probably already familiar with the basic rules. There is a Unix-style terminal at the bottom of the screen that responds to your keyboard and displays messages related to your progress in the game. Most of the screen is taken up by an animated scene that replaces the abstract but morbid visuals that most Hangman games use. My main goal for this layout was to make it resolution independent, and I achieved this with vector graphics and fluid widths on all sections.

    Skills used:

    • HTML
    • SVG
    • JavaScript: jQuery, Velocity.js
    • CSS

  • Aquarium Quiz

    2015
    • Demo
    • Source Code

    This was my first attempt at making a HTML5 canvas game, though it's an incredibly simple one. Kind of like an aquatic Where's Waldo, this game gives you a randomly generated aquarium full of fish to look at for five seconds and asks you a question about it. The game itself isn't actually interactive; the idea is that you (and possibly a friend or two) watch the screen and try to guess close to the actual answer.

    Skills used:

    • HTML5 Canvas
    • JavaScript
