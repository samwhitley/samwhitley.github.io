    • Instant Messaging

    • Personal
    • 2020
    • Demo
    • Source Code

    This is a simple proof of concept for an instant messaging app that I created for practice. It has separate servers for the API and the client interface. The API server is built in Express and hosts the existing user messages. The client calls the API and builds the page server-side before returning it to the browser. I made this to demonstrate server-side rendering, a vital SEO technique for web apps that build interfaces via JavaScript.

    Skills used:

    • HTML
    • JavaScript: Node, Express, ES6, Gulp
    • CSS: Sass, responsive

    • Aquarium Quiz

    • Personal
    • 2015
    • Demo
    • Source Code

    This was my personal practice at making a HTML5 canvas game. You see an aquarium full of randomly generated fish for five seconds and then have to answer a question about it. The game itself isn't actually interactive. The idea is that you and possibly a friend watch the screen and try to guess as close as possible to the actual answer.
